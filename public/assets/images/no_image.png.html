AI Prompt (Final Enhanced Version): Generate the Panasonic ISD Asset Management System Role: You are an expert full-stack developer specializing in creating single-file, production-ready web applications with a world-class, modern UI/UX and seamless Supabase integration. Primary Task: Your task is to generate a complete and robust "Panasonic ISD Asset Management System" as a single HTML file. This application must be a comprehensive, enterprise-grade tool for tracking the entire lifecycle of IT assets. It must be intuitive, secure, and visually polished, with a fluid user experience and an elegant design language reflecting Panasonic's corporate branding. Technology Stack & Constraints: Structure: A single index.html file. Styling: Tailwind CSS, loaded via CDN. Use a professional and elegant color palette (Panasonic blues #00529B, greys, and white). Use the Inter font for clean, modern typography. Icons: Use Lucide Icons (loaded via CDN) for all buttons and navigation links to enhance visual communication. JavaScript: All application logic must be in vanilla JavaScript within <script> tags. Use modern ES6+ syntax (async/await, modules). Backend: Supabase. The application must use the Supabase JS Client (loaded via CDN) for all database operations, authentication, and storage. Do not mock any data; the application must be fully functional. Supabase Backend Schema (Assume these tables exist): users: (Managed by Supabase Auth) Contains user info and a role column (System Admin, IT Staff, Department PIC). assets: id, asset_id, serial_number, model, category, purchase_date, price, warranty_expiry_date, status, assigned_to_user, assigned_to_department, created_at maintenance_logs: id, asset_id, log_date, description, cost, logged_by audit_trail: id, asset_id, timestamp, user_email, action attachments: (For the new Attachments feature) id, asset_id (Foreign Key), file_name (Text), file_url (Text, from Supabase Storage), uploaded_at (Timestamp) Detailed Functional Breakdown with UX & New Feature Enhancements 1. Login Page: Create a custom, minimalist form for email/password login with the Panasonic logo. Include a "Remember Me" checkbox for user convenience. Upon successful login, the app must fetch the user's role and direct them to the appropriate view with a smooth fade-in transition. 2. Main Application Layout: A persistent, collapsible sidebar for navigation with Lucide icons and text labels. A header containing a Global Search Bar (to find any asset by ID, serial number, or model from anywhere in the app) and a user profile dropdown with their name, role, and a logout button. 3. Home Page (Dashboard): KPI Cards: Use elegant cards with icons and smooth hover effects to display: Total Assets, Assets In Use, Assets In Storage, Assets Expiring within 90 days, and Total Asset Value (sum of all asset prices). Charts (use Chart.js or similar): Pie Chart: "Assets by Status". Bar Chart: "Assets by Category". New Line Chart: "Assets Added Over Time (Last 6 Months)" to visualize procurement trends. Recent Activity Feed: A section that shows the last 5-10 actions performed in the system (e.g., "ISD-LP-123 was assigned to John Doe by [User]"), providing immediate context. 4. Asset Registration Page: A clean, multi-step form for adding a single asset to prevent user overwhelm. Autocomplete/Suggestions: For fields like Model and Department, provide suggestions based on existing entries to reduce typing and errors. Implement real-time, client-side validation with clear error messages. Bulk Registration: Provide a downloadable CSV template. When a file is uploaded, show a preview of the data for confirmation before submitting to Supabase. 5. Asset List Pages (In Storage, Assigned, etc.): Use a modern, responsive table design. Advanced Filtering: Include dropdown filters for Category and Status in addition to the search bar. Contextual Status Indicators: Use colored "pills" or dots for the status column (e.g., green for "In Use," yellow for "In Storage," red for "Under Repair") for quick visual scanning. Actions per row: Quick View: An "eye" icon that opens a slide-over panel from the side, showing key asset details without leaving the table view. Print QR Code: A button that opens a clean, printable modal with the QR code and Asset ID. 6. Asset Detail View (Target of QR codes and Quick View): This should be a comprehensive, well-designed modal or slide-over panel. Use a tabbed interface to neatly organize information: Tab 1: Details: All core asset information. Tab 2: Maintenance History: A chronological list of all maintenance logs. Tab 3: Attachments: (New Feature) A section to upload, view, and download related files (e.g., PDF invoices, warranty documents) using Supabase Storage. Tab 4: Audit Trail: The complete, uneditable history of all changes. 7. Lifecycle Planning & Forecasting Module: A dedicated, interactive page. Include a date range selector and a filter by Category. Display results in both a table and a visual timeline chart to make it easy to see upcoming expiry dates. Budget Impact Summary: Automatically calculate and display the estimated budget required to replace the forecasted expiring assets (e.g., "Forecast requires an estimated budget of $5,000 for 5 new laptops"). 8. New Module: Check-out / Check-in for Loaner Equipment: A new page dedicated to managing temporary loans of equipment. Functionality: An IT Staff member can select an asset from "In Storage" and "Check-out" to an employee with an expected return date. The dashboard will have a new KPI card for "Assets on Loan". The system will send a notification when the expected return date is approaching. A simple "Check-in" button returns the asset's status to "In Storage". This is perfect for managing temporary replacements or project-specific hardware. 9. New Module: Supplier & Vendor Management: A simple CRUD module for the System Admin to manage a list of approved suppliers and vendors. Functionality: Store supplier name, contact person, email, and phone number. When registering a new asset, the "Supplier" field will be a dropdown populated from this list, ensuring data consistency. Final UX & UI Instructions (Crucial for a Professional Feel): Feedback & Notifications: Use non-blocking toast/snackbar notifications for feedback (e.g., "Asset successfully created!"). Avoid using native browser alert(). Loading States: Show elegant skeleton loaders or spinners while data is being fetched from Supabase. Never show a blank page. Empty States: When a table has no data, display a helpful message with an icon (e.g., "No assets found in storage. You can register a new asset here."). Smooth Transitions: Use subtle CSS transitions for all state changes, such as modals appearing, sidebars collapsing, and hover effects, to make the UI feel fluid and responsive. Accessibility (A11y): Ensure the application is accessible by using semantic HTML and adding ARIA labels to interactive elements where necessary. The final output must be a single, complete, and production-quality HTML file.
